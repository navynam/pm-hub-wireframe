# PM 업무 통합 관리 플랫폼 - 기능정의서

> **문서 버전:** v1.2  
> **작성일:** 2026-02-23  
> **설계 원칙:** 심플 · 직관 · 실용  
> **대상:** SI/솔루션 중소기업 PM (프로젝트당 10~20명 규모)

---

## 설계 철학

| 원칙 | 설명 |
|------|------|
| **심플 우선** | ERP가 아니다. 클릭 3번 안에 원하는 정보에 도착 |
| **소규모 최적화** | 프로젝트당 10~20명. 수백 명 관리용 복잡한 기능 배제 |
| **실용성** | PM이 실제로 매일 쓰는 기능만. "있으면 좋겠다"는 빼기 |
| **빠른 입력** | 양식 최소화, 자동완성, 기본값 적극 활용 |

### 네비게이션 구조

```
┌─────────────────────────────────────┐
│  📊 대시보드                         │  ← 홈. 모든 것의 요약
│  👤 인력 풀                          │  ← M1
│  🔄 인력 파이프라인                   │  ← M2 핵심. 철수/재배치 한눈에
│  📁 프로젝트                         │  ← M2+M3+M4+M6 통합
│  📝 제안                             │  ← M5
│  ──────────                         │
│  ⚙️ 설정                             │
└─────────────────────────────────────┘
```

> **핵심:** 메뉴 5개 + 설정. 인력 파이프라인은 PM이 매일 보는 화면이라 독립 메뉴. 프로젝트를 클릭하면 그 안에서 인력·일정·비용·계약을 탭으로 전환.

## Module 2: 인력 투입 & 철수/재배치

> 프로젝트에 사람 넣고, 빠질 때 놓치지 않고, 다음 자리를 찾아주는 것. **PM의 가장 핵심 업무.**

### 기능 목록

#### 투입 관리

| ID | 기능명 | 설명 | MVP |
|----|--------|------|-----|
| M2-01 | 인력 배정 | 인력 풀에서 검색 → 프로젝트에 추가. 역할·투입률·기간 설정 | ✅ |
| M2-02 | 투입률 | 50%, 100% 등. 숫자 직접 입력 | ✅ |
| M2-03 | 역할 지정 | PM, PL, 개발, 디자인, QA, 기타 | ✅ |
| M2-04 | 투입 기간 | 시작~종료일 | ✅ |
| M2-05 | 중복 투입 경고 | 같은 사람이 여러 프로젝트에서 투입률 합계 100% 초과 시 경고 | ✅ |
| M2-06 | 투입 인력 목록 | 프로젝트별 현재 투입 인력 테이블 | ✅ |
| M2-07 | 교체/철수 처리 | 인력 교체·철수 처리. 사유 메모 | ✅ |

#### 🔥 철수/재배치 워크플로우 (핵심)

| ID | 기능명 | 설명 | MVP |
|----|--------|------|-----|
| M2-10 | 철수 예정 알림 | 투입 종료일 기준 **D-30, D-14, D-7** 자동 알림. 대시보드 + 벨 알림 | ✅ |
| M2-11 | 대기 예정 인력 리스트 | 향후 30일 내 철수 예정인 인력을 자동 수집하여 리스트 표시. 철수일·현재 프로젝트·기술스택 한눈에 | ✅ |
| M2-12 | 재배치 추천 | 대기 예정 인력의 기술스택 기반으로, 현재 진행 중인 프로젝트·제안 중 매칭되는 건 자동 추천. 매칭 점수(기술 일치도) 표시 | ✅ |
| M2-13 | 인력 파이프라인 뷰 | 전사 인력 타임라인: 누가 언제 어디서 빠지고, 다음 배정이 있는지/없는지를 바 차트로 시각화 | ✅ |
| M2-14 | 빠른 재배정 | 파이프라인 뷰에서 대기 구간을 클릭 → 추천 프로젝트 선택 → 바로 배정 (3클릭 완료) | ✅ |
| M2-15 | 전사 가동률 | 전체 인력 가동률 요약 (대시보드 위젯). 가동률 = 투입인력/전체인력 | ✅ |

### 철수/재배치 워크플로우 흐름

```
투입 종료 D-30 ──→ 🔔 알림: "홍길동, 프로젝트A 철수 예정 (3/25)"
                        │
                        ▼
              📋 대기 예정 인력 리스트에 자동 등록
                        │
                        ▼
              🤖 재배치 추천 자동 생성
                 "홍길동(Java/Spring) → 프로젝트B(Java 개발자 필요) 매칭 87%"
                 "홍길동(Java/Spring) → 제안C(Java SI, 평가중) 매칭 72%"
                        │
                        ▼
              📊 인력 파이프라인 뷰에서 확인
                 홍길동: ██████████░░░░░░░░░░
                         프로젝트A    ⬚ 대기    → 프로젝트B?
                        │
                        ▼
              ✅ PM이 클릭 → 재배정 확정
```

### 재배치 추천 로직

| 매칭 기준 | 가중치 | 설명 |
|-----------|--------|------|
| 기술스택 일치 | 50% | 필요 기술과 보유 기술 겹침 비율 |
| 경력/등급 적합 | 20% | 요구 등급과 인력 등급 일치 여부 |
| 일정 맞춤 | 20% | 철수일과 투입 필요 시작일 갭 (적을수록 높음) |
| 고객사 경험 | 10% | 해당 고객사 프로젝트 경험 여부 |

### 화면

| 화면 | 설명 |
|------|------|
| 프로젝트 > 인력 탭 | 투입 인력 테이블 + 배정 버튼 + 철수 예정 뱃지 |
| 인력 배정 모달 | 인력 풀 검색 → 역할/투입률/기간 입력 → 배정 |
| ⭐ 인력 파이프라인 | 전사 인력 타임라인 바 차트. 대기 구간 하이라이트. 클릭→재배정 |
| ⭐ 대기 예정 리스트 | 향후 30일 내 철수 예정 인력 + 재배치 추천 카드 |
